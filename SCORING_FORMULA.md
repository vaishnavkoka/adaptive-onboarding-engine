# 📊 Scoring Formula Reference\n\n## Overview\n\nThe AI-Adaptive Onboarding Engine uses a **fair, transparent, and deterministic** scoring algorithm. Every score is calculated using the same formula, ensuring consistency and fairness.\n\n---\n\n## The Scoring Formula\n\n### Primary Formula\n\n```\nFinal Score = min(Base% + Proficiency Bonus, 100)\nwhere:\n  Base% = (Matching Skills / Required Skills) × 100\n  Proficiency Bonus = (Number of Proficiency Matches) × 3\n  Minimum Floor = 10 (if any matching skills found)\n```\n\n### Key Components\n\n#### 1. Base Percentage\n```\nBase% = (Matching Skills / Required Skills) × 100\n\nExample:\n  Resume has: Python, Java, AWS\n  Job requires: Java, AWS, Kubernetes, Docker\n  Matching: Java, AWS = 2 matches\n  \n  Base% = (2 / 4) × 100 = 50%\n```\n\n#### 2. Proficiency Bonus\n```\nProficiency Bonus = (Number of Proficiency Matches) × 3 points\n\nWhere \"Proficiency Match\" means:\n  Resume Skill Level ≥ Job Required Level\n  \nExample 1: Resume has \"Java (Expert)\", Job wants \"Java (Intermediate)\"\n  ✅ Proficiency Match (Expert ≥ Intermediate)\n  \nExample 2: Resume has \"Java (Beginner)\", Job wants \"Java (Advanced)\"\n  ❌ No Proficiency Match (Beginner < Advanced)\n  \nWith 2 proficiency matches:\n  Proficiency Bonus = 2 × 3 = +6 points\n```\n\n#### 3. Minimum Floor\n```\nMinimum Floor = 10 points\n\nApplied if:\n  - At least one skill matches resume to job\n  - Prevents extremely low scores when matching exists\n  \nExample: If Base% = 5% but 1 proficiency match:\n  Score = max(5% + 3, 10) = 10%\n```\n\n#### 4. Score Cap\n```\nScore is capped at 100 (realistic matching)\n\nExample: If calculation gives 156%\n  Final Score = min(156%, 100%) = 100%\n```\n\n---\n\n## Worked Examples\n\n### Example 1: Strong Match\n\n**Scenario**: Entry-level job seeker with many matching skills\n\n**Data**:\n- Resume Skills: Python (Expert), JavaScript (Expert), React (Intermediate), AWS (Beginner)\n- Job Required: Python, JavaScript, React, Node.js (all Expert/Intermediate level)\n\n**Calculation**:\n1. **Identify Matches**:\n   - Python: ✅ (has Expert, needs Expert) → Proficiency match\n   - JavaScript: ✅ (has Expert, needs Expert) → Proficiency match  \n   - React: ✅ (has Intermediate, needs Intermediate) → Proficiency match\n   - Node.js: ❌ (not found in resume)\n\n2. **Calculate Base%**:\n   - Matching Skills: 3\n   - Required Skills: 4\n   - Base% = (3 / 4) × 100 = **75%**\n\n3. **Calculate Proficiency Bonus**:\n   - Proficiency Matches: 3 (Python, JavaScript, React all match levels)\n   - Bonus = 3 × 3 = **+9 points**\n\n4. **Final Score**:\n   - Score = 75 + 9 = **84%**\n   - ✅ **Capped at 100**: 84 < 100, so **Final = 84%**\n   - **Interpretation**: 🟢 **Excellent Match**\n\n---\n\n### Bonus = 2 × 3 = **+6 points**\n\n4. **Final Score**:\n   - Score = 100 + 6 = **106%**\n   - ✅ **Capped at 100**: 106 > 100, so **Final = 100%**\n   - **Interpretation**: 🟢 **Excellent Match** (Perfect!)\n\n---\n\n## Score Interpretation Scale\n\n| Score Range | Color | Meaning | Learning Path |\n|-------------|-------|---------|----------------|\n| **80-100%** | 🟢 | Excellent Match | Minor simple and explainable\n   - Users understand exactly how their score is calculated\n   - Every number is visible in the breakdown\n\n2. **Deterministic**: Same inputs always produce same output\n   - No randomness or hidden bias\n   - Consistent across analyses\n\n3. **Rewarding**: \n   - Proficiency matching gives bonus points\n   - Encourages depth over breadth\n   - Acknowledges transferable skills\n\n4. **Realistic**:\n   - Minimum floor prevents unfair low scores on partial matches\n   - Score cap at 100 is realistic\n   - Proportional to actual matching\n\n5. **Inclusive**: Works with any job category and skill level\n   - No hardcoded assumptions\n   - Scales with number of skills\n   - Fair to junior and senior candidates\n\n### 🎯 Design Decisions\n\n**Base % Formula** (Matching / Required × 100)\n- Reflects actual skill match percentage\n- Fair: measures job requirement coverage\n- Intuitive: 50% means \"have half the skills\"\n\n**Proficiency Bonus** (+3 pts per match)\n- Rewards having the right level\n- Not too high: max bonus ~18 pts on large jobs\n- Balanced: recognizes depth without dominating\n\n**Minimum Floor** (10 pts)\n- Prevents \"lucky guesses\" (matching 1 rare skill gives bonus)\n- Encourages meaningful matches\n- Fair to career changers with some overlap\n\n**Score Cap** (100 pts)\n- Realistic: can't be \"more perfect\" than perfect\n- Fair: doesn't artificially inflate overqualified candidates\n- Psychological: 100% is understandable maximum\n\n---\n\n## Proficiency Levels Explained\n\n### Standard Levels\n\n| Level | Definition | Example |\n|-------|-----------|----------|\n| **Expert** | 5+ years or advanced knowledge | \"Python: Expert\" |\n| **Intermediate** | 2-5 years or solid working knowledge | \"Java: Intermediate\" |\n| **Beginner** | <2 years or basic knowledge | \"Kubernetes: Beginner\" |\n| **Mentioned** | Mentioned but proficiency unclear | Generic skill mention |\n\n### Matching Rules\n\n```\nFor a proficiency match:\n  Resume Proficiency Level ≥ Job Required Level\n  \nComparison Order:\n  Expert ≥ Intermediate ✅\n  Expert ≥ Beginner ✅\n  Intermediate ≥ Beginner ✅\n  Intermediate ≥ Expert ❌\n  Beginner ≥ Intermediate ❌\n  Beginner ≥ Expert ❌\n```\n\n### Extraction Process\n\nProficiency levels are determined by:\n- **Manual text analysis**: \"5 years Python\" → Expert\n- **Keyword detection**: \"Python expert\" → Expert\n- **Context**: \"Familiar with Docker\" → Beginner\n- **LLM analysis**: Understanding context in full text\n\n---\n\n## Score Breakdown Components\n\n### Display Details\n\n### Code Location\n\n**File**: `onboarding_engine.py`  \n**Method**: `calculate_score()`\n\n```python\ndef calculate_score(your_skills, required_skills):\n    \"\"\"\n    Fair scoring formula:\n    Base% = (Matches / Required) × 100\n    Bonus = Proficiency Matches × 3\n    Floor = 10 (if any matches)\n    Final = min(Base + Bonus, 100)\n    \"\"\"\n    # Calculate matching skills\n    matches = len([s for s in your_skills if s in required_skills])\n    \n    # Calculate base percentage\n    base_percent = (matches / len(required_skills) * 100) if required_skills else 0\n    \n    # Calculate proficiency bonus\n    proficiency_matches = sum(1 for s in your_skills \n                             if s in required_skills and \n                                levels[s] >= job_levels[s])\n    bonus = proficiency_matches * 3\n    \n    # Combine with minimum floor\n    score = max(base_percent + bonus, 10) if matches > 0 else 0\n    \n    # Cap at 100\n    return min(score, 100)\n```\n\n---\n\n## Customization Guide\n\n### Changing the Bonus Amount\n\n**Current**: +3 points per proficiency match  \n**To Change**: Edit line in
